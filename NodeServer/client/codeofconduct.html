Personal coding guidelines:

Single Responsibility:
This means that no class/function/method or module should do more than one thing.
It also implies that an object is resonsible for it's own data and for creating/disposing of itself.

Self documenting code:
Naming variables, properties, fields, classes, modules, methods and functions so that the name tells the reader what it does or is.

Open-Closed:
Open for extension, Closed for modification.
Only expose what is absolutely necessary. Desig your code to be extendable.
